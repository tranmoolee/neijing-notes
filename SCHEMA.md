# SCHEMA.md — Agent 读取规范

> 本文件供 Agent 在执行发布任务前读取。
> 描述各 Markdown 文件的结构、字段含义、以及图片/音频/视频的命名规则。

---

## 一、article.md 结构

每篇 `article.md` 是博客正文,结构固定如下:

```
# 标题

> 免责声明(固定段落)

---

## 今天这段          ← 引用的原文段落
## 第一遍读          ← 作者第一直觉
## [卡住的词/句]     ← 深挖某个概念(可能有多个二级标题)
## 这一段暂时的理解  ← 大白话总结,标注"未必对"
## 留的坑            ← 未解决的问题编号列表

---
[下篇预告]
```

**Agent 注意**:
- `## 今天这段` 下的引用块(`> `) 是原文,不可修改
- `## 留的坑` 是编号列表,格式为 `1. **问题** 说明`
- 文章末尾有下篇预告,可提取作为下期推送文案

---

## 二、publish.md 结构

每篇 `publish.md` 包含四个部分,用 `---` 分隔:

```
# 标题 —— 配图 Prompt + 小红书图文版

---

## Part 1 · 文生图 Prompt(共 N 张)

### Prompt 1 · [用途说明]
> 用途:...

**生图描述:**
[代码块:喂给 AI 生图工具的 prompt]

**封面叠加文字:**
[代码块:设计工具中叠加的文字内容,含层级/字号建议]

---

## Part 2 · 小红书图文版(9 图卡片)

### 📌 封面卡
### 📌 卡 1 ~ 卡 8
### 📌 下期预告卡

### 📝 小红书发布正文
[代码块:配图下方的正文]

### 🏷️ 推荐标签
[代码块:标签列表]

---

## 📐 排版说明
[字号建议表格 + 重点卡说明]
```

**Agent 执行发布任务时的提取规则**:

| 任务 | 提取字段 |
|---|---|
| 生成配图 | `## Part 1` 下每个 `**生图描述:**` 代码块 |
| 小红书排版 | 每张卡的 `**标题:**` + `**正文:**` + `**叠加文字:**` |
| 发小红书 | `### 📝 小红书发布正文` 代码块 + `### 🏷️ 推荐标签` |
| 博客发布 | 读取同目录 `article.md` 全文 |

## 三、images/ 命名规则

| 文件名 | 内容 |
|---|---|
| `cover.jpg` | 封面图(对应 Prompt 1) |
| `cover-alt.jpg` | 封面备选图(对应 Prompt 1B,如有) |
| `card-01.jpg` | 小红书卡片 1 |
| `card-02.jpg` | 小红书卡片 2 |
| … | … |
| `card-09.jpg` | 小红书卡片 9(下期预告) |
| `blog-hero.jpg` | 博客头图(通常与 cover 同源,比例 16:9) |

所有图片分辨率建议:
- 小红书卡片:`1080×1440px`(3:4)或 `1080×1080px`(1:1)
- 博客头图:`1920×1080px`(16:9)
- 封面在缩略图(`200×267px`)状态下主标题必须可读

---

## 四、audio/ 命名规则

| 文件名 | 内容 |
|---|---|
| `podcast.mp3` | 播客完整版(朗读 article.md 全文) |
| `voiceover.mp3` | 配视频用旁白(精简版,约 2-3 分钟) |
| `reel-audio.mp3` | 短视频配音(≤60 秒) |

---

## 五、video/ 命名规则

| 文件名 | 内容 |
|---|---|
| `xiaohongshu.mp4` | 小红书视频版(9:16,≤3 分钟) |
| `bilibili.mp4` | B站版本(16:9,完整版) |
| `reel.mp4` | 竖版短视频(9:16,≤60 秒) |

---

## 六、新增一篇的标准流程

1. 在根目录新建文件夹,格式:`NN-主题关键词`(NN 为两位数编号)
2. 在文件夹内创建:`article.md` `publish.md` `images/` `audio/` `video/`
3. 在 `README.md` 的系列目录表格里新增一行,状态改为 `🔲 待写`
4. 写完后状态改为 `✅ 已完成`
5. git commit,push 到 main 分支

---

## 七、Agent 执行配图任务的完整步骤

```
1. 读取目标篇的 publish.md
2. 提取 Part 1 下所有 **生图描述:** 代码块
3. 调用图像生成 API(Flux / SDXL / Midjourney)
4. 生成图片后按命名规则存入 images/
5. 读取每张卡的 **封面叠加文字:** 或正文
6. 调用设计工具 API 或本地脚本叠加文字
7. 导出最终图片,覆盖 images/ 对应文件
8. git add images/ && git commit && git push
```
