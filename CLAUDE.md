## 项目简介

博客系列《学渣读内经》的内容仓库。每篇对应一个独立文件夹，包含博客正文、小红书发布素材和配图。

- GitHub：https://github.com/tranmoolee/neijing-notes（tranmoolee 账户，main 分支）
- 博客主站：https://www.ileemoo.com

## 每篇文件夹结构

```
NN-主题/
├── article.md            ← 博客正文（母稿）
├── publish.md            ← 小红书图文 + 配图 Prompt + 标签
├── images/               ← 配图存放位置
├── audio/                ← 音频（播客/语音稿）
├── video/                ← 视频（剪辑素材/成片）
├── ghost.md              ← Ghost 博客发布稿（含 frontmatter）
├── wechat.md             ← 微信公众号发布稿
├── wechat.publish.html   ← 公众号排版 HTML
├── x.md                  ← X/Twitter 线程稿
├── xhs-publish-pack/     ← 小红书手机手发包（图片+文案）
└── publish-log.json      ← 各平台发布状态记录
```

## 新增一篇的标准流程

按 `SCHEMA.md` 第六节执行：

1. 新建 `NN-主题/` 文件夹（NN 为两位数编号）
2. 创建 `article.md` `publish.md` `images/` `audio/` `video/`
3. 在 `README.md` 系列目录新增一行，状态 `🔲 待写`
4. 写完后状态改为 `✅ 已完成`
5. git commit + push

**提交格式**：`feat: NN-主题 article + publish`

## 平台稿与合规（公众号/Ghost）

- `wechat.md`（公众号）和 `ghost.md`（Ghost）的 **markdown 正文，由 `article.md` 经根目录脚本 `.gen-platform.py` 自动派生**——两端正文保持一致（"Ghost 跟随公众号"），ghost.md 多一段 frontmatter。改了 `article.md` 后重跑 `python3 .gen-platform.py [文件夹...]` 即可同步；新增篇目要先在该脚本的 `DATA` 字典里登记 slug/tags/excerpt。
- 派生稿文末统一带 **AI 标识 + 健康免责 footer**（"由作者撰写、AI 辅助整理与排版……不构成就医建议"），这是规避公众号"低创作度内容"判定的**合规要求，必须保留**。
- 实际发布（上传 Ghost/微信草稿、叠字图等）见 `WORKFLOW.md`；`.gen-platform.py` 只生成 markdown 稿件，不发布。

## 写作"人味"要求（避免被判低创作度）

中医/健康属公众号敏感领域，纯"AI 讲解器"式内容会被限流。每篇须：① 有真实生活锚点（西班牙生活、父亲扎针的临床观察、熬夜/久坐等亲身体感）；② 打散固定模板（别每篇都"第一遍读→卡住的词"）；③ 比喻不要每篇雷同。**只写生活与体感，绝不写疗效、不写"治好了"。**

故事化写作规范见根目录 `CHUANMU-故事化写作协议-v1.md`（选题反差切口、六段骨架、证据规则、发布前自检 5 问）。
